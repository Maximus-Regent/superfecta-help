# Validation Quickstart

This is the shortest practical guide for re-checking the project after edits.

Use the **smallest validator that matches the change**.
If the change was broad or touched both research logic and paper-trade operations, use the top-level project sweep.

Evidence boundary for this runbook:
- valid_evidence_scope=validation_quickstart_navigation_contract_only
- Valid use: validator selection, read order, output-path checks, and parent-rollup reuse guardrails only.
- Not settled ROI, live profitability, promotion readiness, bankroll guidance, or real-money evidence.
