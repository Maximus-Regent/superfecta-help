CD_CORE_K8 was disqualified in that fold at pos_year_ratio=0.46, just 4pp below threshold, while delivering +45.65% test ROI on 41 races.

### 2026-04-16 01:15 EDT — Downstream A/B comparison: baseline vs enriched horse-history model through EV engine
- What changed: built `ab_downstream_comparison.py`, an honest downstream comparison that tests whether the enriched horse-history model's improved payout prediction accuracy translates to better EV-engine betting decisions. The script loads race data, reproduces the chronological 75/25 split, builds features for both models, scores every winning combo in the 22,244-race test set with both models, then applies the EV ticket engine's conservative thresholds (75% haircut, 15% min ROI, Kelly > 0) to each prediction.
- First run (unfair): initially compared the `full_data_retrain_model.json` (552K races, all tracks) against `full_data_retrain_horse_model.json` (88K races, major tracks). The baseline appeared slightly more accurate (RMSE 0.4065 vs 0.4115) — but this was an unfair comparison because the baseline had 6x more training data. Identified this as a confound and discarded the result.
- Fair run: trained two matched models on the *same* dataset (14years_major_tracks.csv, 88K races) — `ab_baseline_model.json` (52 features) and `ab_enriched_model.json` (172 features, with horse-history). Both use the same chronological 75/25 split, same hyperparameters, same outlier removal.
- Results (fair comparison, 22,244 out-of-sample races):
  - **Prediction accuracy**: Enriched wins. Log-ratio RMSE 0.4115 vs 0.4205 (**-2.2%**), R² 0.6186 vs 0.6016 (**+1.7pp**), payout RMSE $5,929 vs $6,192 (**-$262**). Improvement is consistent across all percentiles (p10 through p90 of absolute payout error).
  - **EV engine pass-through**: Nearly identical. Baseline passes 178 winning combos (0.80%), enriched passes 171 (0.77%). The enriched model is slightly more conservative — better accuracy means fewer extreme predictions that accidentally clear the threshold.
  - **Disagreement analysis**: 108 winners pass both filters; 70 pass only baseline; 63 pass only enriched; 22,003 pass neither. The enriched-only bucket has marginally higher average payout ($489/winner vs $452/winner for baseline-only), but on ~65 races this is noise.
  - **Net effect on downstream betting**: Negligible. The EV engine is so conservative (only 0.8% of winning combos pass) that the prediction improvement doesn't meaningfully change which bets get placed.
- Limitation (stated clearly in the artifact): this comparison tests predictions on *actual winning combos* only. It does not test whether the enriched model changes the *ranking* of non-winning combos, which would require integrating horse-feature lookup into `score_race_combos` — a larger infrastructure change.
- What was learned: the enriched model's prediction improvement is real and survives on the fair out-of-sample test. But it does NOT translate to meaningfully different downstream betting behavior. The EV engine's conservative thresholds dominate: the 75% haircut and 15% min ROI filter wash out the modest prediction gain. This is consistent with the project's existing finding (COLE_STATUS_AND_PLAN.md) that "the XGBoost residual model produces zero improvement over simple filters" — the betting edge comes from Phase 7 gap-filter rules, not from payout prediction accuracy.
- Decision: the enriched horse-history model should **NOT replace the production model yet**. The prediction improvement is genuine but the downstream evidence is neutral. Replacing the production model would also require integrating horse-feature lookup into the live prediction pipeline (`predict_single_race.py`, `model_main.py`), which is a non-trivial change for negligible betting benefit.
- Artifacts: `ab_downstream_comparison.py` (comparison script), `ab_downstream_comparison_results.json` (full results), `ab_baseline_model.json` / `ab_enriched_model.json` (matched models for fair comparison), `ab_baseline_results.csv` / `ab_enriched_results.csv` (per-race predictions), diagnostic plots.
